Fase 2 - Sensor Ultrassônico - Teste de Software

Objetivo
Testar o projeto e identificar os dois defeitos introduzidos. Os defeitos podem ser encontrados no código, diagramas e esquemático. Para cada defeito encontrado, apresentar um caso de testes que revele o defeito.

Descrição da Fase
A fase 2 consiste em representar um sensor de distância que, ao aproximar um objeto do sensor ultrassom, o buzzer irá apitar, e ao afastar ficará em silêncio. Quando aproximado e a distância entre o objeto e o sensor ultrassom for menor que 50cm, o buzzer começará a apitar proporcionalmente à essa distância, diminuindo o intervalo entre apitos conforme a distância é diminuída. Ao afastar o objeto do sensor ultrassom, conforme o objeto for ficando mais distante do sensor, o buzzer irá aumentar o intervalo entre os apitos, e assim que essa distância for maior que 50cm, o buzzer ficará em silêncio.

Itens Utilizados
1 - Arduino.
1 - Sensor Ultrassom.
1 - Buzzer.
Jumpers.
1 - Protoboard.

Esquemático
Utilize o esquemático abaixo para montar o circuito.

Código
#include < Ultrasonic.h >

#define BUZZER 5
#define multiplicadorTempo 20 //cm*multiplicador = intervalo entre bips

Ultrasonic ultrasonic(12,11); // (Trig PIN,Echo PIN)

int distancia;

void setup() {
}

void loop()
{
  distancia = ultrasonic.Ranging(CM);
  delay(50);        //intervalo para leitura do sensor
     
  tone(BUZZER, 2650, 100); //2650Hz 100ms
  delay(distancia*multiplicadorTempo);
}
Casos de Teste
O Teste 1 revela o seguinte defeito: O buzzer apita o tempo todo, independente da distância.
Caso de Teste
ID
 1
Objetivo

Ator
 Aluno de Engenharia de Software
Pré-condições
 Objeto colocado a uma distância maior que 50 cm do sensor ultrassom
Procedimentos (Entradas e saídas)
 1 – Aproximar o objeto do sensor lentamente

2 – O intervalo entre os bips deve diminuir conforme a distância também diminui
Pós-condições
 Bip contínuo quando objeto a menos de 1 cm do sensor
